How it works

A traceable workflow that converts dense research into a listener-ready story without compromising rigor or provenance. Each step produces artifacts you can inspect.

01

Upload and ingest

Each PDF is parsed page-by-page and normalized for clean extraction. Citations are preserved so every statement stays linked to the original evidence.

02

Explain in daily-life terms

The script writer translates methods and results into plain-language analogies. Citations stay attached to every segment so reviewers can validate the narrative.

03

Deliver polished audio

Narration pacing adapts to cognitive load to keep dense material clear. The final mix is normalized with transcripts and quality metadata.

Audio engineered for clarity

The pacing model reads math density, lexical variety, and sentence length to set speaking rate and pauses. Output is loudness-normalized, QA-scored, and exported alongside transcripts and metadata for verification.

Developer-friendly by default

The API submits jobs, streams progress over SSE, and writes deterministic artifacts to disk. The worker runs a clean-architecture pipeline (extract → chunk → script → optional retrieval → TTS → audio assembly), while adapters keep LLMs, embeddings, and TTS providers swappable without touching the core domain logic.
